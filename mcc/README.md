# ⚔️ MCC (Maple Combat Calculator)

**MCC**는 본 프로젝트의 심장부에 해당하는 핵심 수식 라이브러리입니다. 메이플스토리의 복잡한 전투 스탯과 스킬 데미지 공식을 C++로 구현하여, 시뮬레이터(MCM)나 외부 도구에 정확한 기대 데미지 결과값을 제공합니다.

## 🚀 주요 역할
*   **전투력 산출**: 공식 전투력 공식을 기반으로 캐릭터의 성장을 정량화합니다.
*   **스킬 데미지 시뮬레이션**: 방어율 무시(방무), 속성 반감, 레벨 차이 보정 등이 모두 반영된 최종 데미지를 계산합니다.
*   **맥뎀 기댓값 보정**: 데미지가 맥시멈 제한(Damage Cap) 근처에 걸쳐있을 때, 손실되는 딜량을 통계적으로 계산하여 정확한 기댓값을 산출합니다.

## 🛠 기술 스택
*   **Language**: C++20 (표준 라이브러리 및 수학 함수 활용)
*   **Data Format**: [Google Protocol Buffers](https://protobuf.dev/) (`shared/proto/internal/mcc_stat.proto`)
    *   스탯 데이터의 직렬화 및 컴포넌트 간 규격 통일을 위해 Protobuf를 사용합니다.

## 📐 핵심 로직 가이드

### 1. 스킬 데미지 공식 (`calcSkillDamageRaw`)
메이플스토리의 일반적인 데미지 공식을 충실히 따릅니다.
*   **기본**: `(주스탯*4 + 부스탯) * 0.01 * 공격력 * 무기상수`
*   **배율**: `(1 + 데미지% + 보공%) * (1 + 최종데미지%)`
*   **몬스터 상호작용**:
    *   `방어율 보정 = max(0, 1 - 몬스터방어율 * (1 - 방무%))`
    *   `속성 보정 = 1 - 몬스터속성내성 * (1 - 속성내성무시%)`
    *   `레벨 보정 = 캐릭터와 몬스터의 레벨 차이에 따른 가감산`

### 2. 맥뎀 보정 로직 (Internal Exact Bivariate Model)
메이플스토리의 데미지는 숙련도(Mastery)와 크리티컬 데미지 범위에 따라 **편차**를 가집니다. 특정 타격이 맥뎀($D$)을 초과할 경우, 해당 초과분은 소실됩니다.

MCC는 이를 수학적으로 가장 엄밀하게 계산하기 위해 **이변수 곱 분포(Bivariate Product Distribution)** 모델을 사용합니다.
*   **변수**: 숙련도 $X$와 크리티컬 데미지 $Y$의 독립적 곱 ($Z = X \cdot Y$)
*   **계산**: $E[\min(Z \cdot maxDamageVal, D)]$ (이변수 적분 기댓값)
*   **특징**: 
    *   **수치적 안정성**: `Boost.Multiprecision` 라이브러리를 사용하여 로그(`log`) 및 제곱 연산 시 발생하는 부동소수점 오차를 완벽히 제거했습니다.
    *   **정밀도**: 50자리 유효숫자를 기반으로 연산하여, 경(京) 단위 이상의 대형 숫자에서도 음수 버그나 오버플로우가 발생하지 않습니다.
    *   **캡슐화**: 해당 로직은 `calcSkillDamageRaw` 내부에 통합되어 외부 노출 없이 안전하게 보호됩니다.

## 📂 디렉토리 구조
*   `inc/`: 헤더 파일 (`.h`)
    *   `combat_power.h`: 전투력 관련 함수 정의
    *   `skill_damage.h`: 스킬 및 도트 데미지 계산 함수 정의
*   `src/`: 구현 파일 (`.cpp`)
    *   `combat_power.cpp`: 직업별 주스탯 매핑 및 공식 구현
    *   `skill_damage.cpp`: 수치 안정성이 보강된 데미지 엔진 구현

## ⚠️ 주의 사항
*   **정밀도**: 본 엔진은 수조 단위 이상의 큰 숫자를 다룹니다. `int` 범위를 쉽게 초과하므로 반드시 `long long` 또는 `double`을 사용하고, 중간 계산 과정에서의 정밀도 손실에 유의하십시오.
*   **의존성**: 빌드 시 Protobuf 컴파일러(`protoc`)가 필요합니다. `vcpkg`를 통해 의존성을 관리하는 것을 권장합니다.
